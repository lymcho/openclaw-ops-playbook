# Contributing to OpenClaw Ops Playbook

Thanks for your interest in improving the playbook! Whether it's a new template, a better pattern, or a typo fix — contributions are welcome.

## Ways to Contribute

- **New templates** — Add a template for a domain or use case not yet covered
- **Template improvements** — Better structure, more "How This Works" sections, clearer placeholders
- **Playbook patterns** — Document new patterns or cross-referencing strategies
- **Bug fixes** — Broken links, formatting issues, typos
- **Site improvements** — Better landing page, accessibility, SEO

## How to Submit

1. Fork this repo
2. Create a branch: `git checkout -b my-improvement`
3. Make your changes
4. Submit a pull request with a clear description of what you changed and why

## Template Guidelines

If you're adding or editing a template:

- Use `.template.md` suffix for all template files
- Include a `## How This Works` section that teaches the assistant how to interact with the data
- Use `[PLACEHOLDER]` for user-specific values
- Add `→ See ...` cross-references where files relate to each other
- Keep it opinionated — templates that make decisions are more useful than blank forms

## Code of Conduct

Be kind. Be constructive. We're all building something useful here.

## Questions?

Open an issue. That's what they're for.
