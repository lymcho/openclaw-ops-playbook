Architecture

Six layers. One system.

Start at the top and work down. Even the first two layers will dramatically improve your assistant.

Bootstrap
First-run identity discovery — your assistant picks its name, creature type, and vibe
Foundation
Agent OS, personality, values, session protocol, memory rules, and safety guardrails
Context
User profile and domain routing — who you are and how work vs. home is separated
Memory
Daily notes, long-term curated memory, and heartbeat state for session continuity

Automation
Heartbeats, departure reminders, CRM nudges, scripts, and extensible skills

Features

The agent OS underneath it all

Battle-tested patterns from daily use. Each layer works alone or as part of the full stack.

🎭

Agent Identity

A bootstrap conversation lets your assistant discover its own name, creature type, and communication style.

🔄

Domain Routing

One assistant, eight domains. ROLES.md routes context by trigger words so your grocery list never leaks into a standup.

📚

Persistent Memory

Daily session notes, long-term curated memory, and heartbeat state. Remembers what matters across sessions.

🔔

Proactive Automation

Departure reminders with real-time driving ETA, CRM nudges, appointment alerts, email triage, weather checks.

🔗

Cross-Referencing

Files link with → pointers. One source of truth, many references — a shared school file serves multiple kids.

💬

Group Chat Aware

Knows when to speak and when to stay quiet. Responds when mentioned or adding value, stays silent for banter.

🛡️

Safety Guardrails

No exfiltration, no destructive commands without asking, prefers trash over rm. Built in from day one.

📱

Platform Formatting

Auto-adapts for Discord, WhatsApp, and chat. No markdown tables in mobile, headers adjusted per platform.

🛠️

Scripts & Skills

Driving ETA via Google Directions API, audio transcription, gog CLI for Gmail & Calendar. Extensible by design.

Quick Start

Up and running in 5 minutes

No dependencies. No build step. Just Markdown files and your OpenClaw workspace.

1

Clone the templates

Copy the templates/ folder into your OpenClaw workspace. Rename files to drop the .template suffix.

2

Bootstrap your assistant

Open BOOTSTRAP.md and start a conversation. Your assistant discovers its own name, creature type, and emoji.

3

Add your context

Fill in USER.md with your details and ROLES.md to define your work/home boundaries.

4

Pick your domains

Add domain files you need — family, household, work, health, finances, travel, hobbies, people. Start with one or two.

5

Wire up automation

Enable heartbeats, departure reminders, and scripts. Your assistant is now proactive, not just reactive.
